// 04 — Documentation

Novagram learning journal

A structured 8-module curriculum built on the real Novagram codebase. Every concept tied to production code, real incidents, and break/fix scenarios.

Module 01
Docker & Containers
Layers, build cache, multi-stage builds, .dockerignore, npm ci, non-root users, Netflix-scale image practices.
Layers Multi-stage Security
Module 02
Networking & VPC
VPC design, CIDR, IGW, route tables, public vs private subnets, security groups, NAT gateway trade-offs, AZ resilience.
VPC CIDR Security Groups
Module 03
ECS Fargate & Task Definitions
Cluster, service, task definition, task — how they compose. Fargate vs EC2, desired count, deployment circuit breakers.
ECS Fargate Circuit Breaker
Module 04
Load Balancing & Routing
ALB listeners, target groups, path-based routing, sticky sessions, WebSocket connection lifecycle during deployments.
ALB Sticky Sessions WebSocket
Module 05
IAM & Secrets Management
Execution role vs task role, OIDC authentication flow, least-privilege policies, SSM Parameter Store as CI/CD bridge.
IAM OIDC SSM
Module 06
CI/CD Pipelines
Pipeline stage design, OIDC permissions, immutable tagging, Trivy CVE policy, concurrency groups, deployment verification.
CI/CD Trivy OIDC
Module 07
Observability
Three pillars — logs, metrics, traces. Four golden signals. Structured JSON logging. CloudWatch Logs Insights. Alert design.
Metrics Logs Traces
Module 08
Scaling Patterns
Auto scaling types, cooldown periods, graceful shutdown, WebSocket scale-in problem, Redis Pub/Sub as the real solution.
Auto Scaling Redis SIGTERM

Kubernetes & Helm
Migrating Novagram from ECS to EKS. Pods, deployments, services, ingress, Helm charts, HPA.
Kubernetes Helm EKS
